Tier 2: Medium Effort (1-3 days each)

5. Add xsect.c Module (Highest Priority New Module)

xsect.c contains the geometry functions that EVERY hydraulic
computation depends on:

  xsect_getAofY(xsect, y)    — area at depth y
  xsect_getWofY(xsect, y)    — top width at depth y
  xsect_getRofY(xsect, y)    — hydraulic radius at depth y
  xsect_getYofA(xsect, a)    — depth at area a (inverse)
  xsect_getSofA(xsect, a)    — section factor

For circular pipes, these are the classic geometric formulas:
  A = (D²/4)(θ - sinθcosθ)   where θ = arccos(1 - 2y/D)
  W = D·sinθ
  R = A / (D·θ)

Showing these across 8 languages would be both:
  - Highly educational (geometry + hydraulics)
  - Practically useful (engineers implement these constantly)

Time: 1-2 days
Impact: +2 points

7. Add controls.c Module

SWMM5's control rules are unique — a domain-specific language
embedded within an engineering model:

  RULE R1
  IF NODE J5 DEPTH > 4.5
  THEN PUMP P1 STATUS = ON

  RULE R2
  IF NODE J5 DEPTH < 1.0
  AND SIMULATION TIME > 01:00:00
  THEN PUMP P1 STATUS = OFF

Translating this rule parser across 8 languages would showcase:
  - Pattern matching (Rust, Julia)
  - String parsing (all languages)
  - State machines (Go, Zig)
  - Interpreter design (Python, JavaScript)

Time: 2-3 days
Impact: +1 point
